USDX PROTOCOL // SHARE-BASED ACCOUNTING

alex@alexandros-securities.com

The USDX protocol implements a rebasing ERC-20 token + share-based accounting to maintain a stable $1 USD peg while enabling closed-end fund mechanics and yield distribution from portfolio strategies. User balances are stored as shares and converted to tokens through a daily-adjusted reward multiplier.

1 USDX = 1 USD
AUM = Token Supply
USDX Balance = shares × rewardMultiplier
balanceOf(account) = shares[account] * rewardMultiplier

When yield accrues from the underlying portfolio (via interest, asset appreciation, etc.), the Oracle role calls addRewardMultiplier to update the conversion rate. This increases all token balances proportionally while keeping share counts constant, reflecting the earned yield without requiring individual transfers.

Portfolio → Income/Return → Oracle Update → Multiplier Adjustment → Token Rebase

The system optimizes gas efficiency by updating a single multiplier value rather than processing separate mints and transfers. All transfers automatically handle the share-to-token conversion, maintaining accurate balances while preserving the 1:1 USD peg through verified reserve backing.
