    <div class="panel">

            <p><strong>USDX TOKEN</strong><br>The USDX Token is an ERC20 compliant fungible token. It implements administrative minting, burning, and rebasing functions to manage the supply of the token.</p>
            <p><strong>WRAPPED USDX TOKEN</strong><br>The Wrapped USDX Token is an ERC4626 compliant fungible token vault. USDX is deposited into the Wrapped USDX smart contract in order to facilitate interoperability and composability with common blockchain applications and accounting systems. Users can withdraw USDX from the Wrapped USDX contract at any time.</p>
            <p><strong>RESERVE VAULT</strong><br>The Reserves Vault is an onchain account holding the onchain portion of the assets backing the value of USDX including yield generating tokenized assets. The Reserves Vault will primary invest in short duration US government securities directly or indirectly.</p>

              <li><strong>DISTRIBUTION TYPES</strong></li>
              <li><strong>Accrual / Accumulating</strong></li>
              <li>1. Yield generated from underlying portfolio</li>
              <li>2. Token price increases (e.g., 1.00→1.01)</li>
              <li>3. Token supply stays fixed</li>
              <li>4. Total value increases (price * supply)</li>
              <li><strong>Rebasing</strong></li>
              <li>1. Yield generated from underlying portfolio</li>
              <li>2. More tokens minted (e.g., 100 tokens → 101 tokens)</li>
              <li>3. Token price stays pegged at $1.00</li>
              <li>4. Total value increases (price * supply)</li>
          </ul>

    <div class="panel">
        <div style="overflow-x: auto;">
            <p><strong>UPDATING THE REWARD MULTIPLIER</strong></p>
            <p>1 Yield accrues intraday; PM submits EOD yield data<br>
               2 Oracle verifies data and portfolio; contract validates yield<br>
               3 addRewardMultiplier applies daily yield, increasing rewardMultiplier<br>
               4 rewardMultiplier multiplies stored shares to compute effective USDX balance<br>
               5 Transfers convert USDX to shares (e.g., 100 USDX = 100/1.047 shares)<br>
               6 Total supply adjusts as totalShares × rewardMultiplier, optimizing gas
            </p>
            <p><strong>DATA FLOW</strong></p>
            <p>1 Portfolio calculates yield and triggers addRewardMultiplier w/ positive increment<br>
               2 addRewardMultiplier verifies increment > 0 and updates rewardMultiplier by adding the increment<br>
               3 _setRewardMultiplier emits the new multiplier, affecting share-to-token conversion<br>
               4 Mint/transfer functions use convertToShares: shares = (amount × BASE) / rewardMultiplier<br>
               5 User balance = shares × rewardMultiplier / BASE, reflecting accrued yield<br>
               6 Transfers update _shares and emit events, preserving yield adjustments<br>
               7 Pausable and blocklist hooks ensure only valid transfers proceed</p>
            <div class="mermaid">

    <button class="accordion2">PAYMENTS</button>
    <div class="panel">
        <div style="overflow-x: auto;">
            <ul>
                <li><strong>STABLECOIN PAYMENT</strong></li>
                <li><strong>Authorization</strong></li>
                <li>1. Sender initiates a transfer via their wallet.</li>
                <li>2. Transaction is broadcast to the blockchain network.</li>
                <li>3. Network verifies sender’s balance and transaction validity.</li>
                <li>4. Transaction is approved and added to the blockchain.</li>
                <li><strong>Processing</strong></li>
                <li>5. Transaction is processed on the blockchain.</li>
                <li>6. Smart contracts execute any required conditions.</li>
                <li>7. Network nodes confirm the transaction.</li>
                <li><strong>Settlement</strong></li>
                <li>8. Stablecoin is credited to the recipient’s wallet.</li>
                <li>9. Transaction is permanently recorded on the blockchain.</li>
